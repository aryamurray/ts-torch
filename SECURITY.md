# Security Policy

## Supported Versions

Security fixes are applied to the `main` branch and the latest published package versions.

Older releases may not receive security patches.

## Reporting a Vulnerability

Please report vulnerabilities privately through GitHub Security Advisories:

1. Open the repository on GitHub
2. Go to the **Security** tab
3. Click **Report a vulnerability**

If you cannot use GitHub Security Advisories, open a GitHub issue and mark it as a security concern without posting sensitive details.

## Response Expectations

- Initial response target: within 3 business days
- Triage and severity assessment: within 7 business days
- Fix timeline depends on severity and exploitability

## Scope

This policy covers:

- Published npm packages in this monorepo
- Build and release workflows
- Native binary loading and platform packages
